Tanulás

Kínai maradéktétel

matematika számelmélet algebra

Legyen \(m\) és \(m_1, m_2, \dots, m_n\) olyan pozitív egész számok, amikre fennáll, hogy \(m=\prod_{i=1}^{n}m_i\), illetve hogy \( \forall (i \neq j): \gcd(m_i, m_j)=1 \).

Tekintsük a következő szimultán kongruenciarendszert

\(x \equiv c_1 \pmod{m_1}\)
\(x \equiv c_2 \pmod{m_2}\)
\(\dots\)
\(x \equiv c_n \pmod{m_n}\)

A kínai maradéktétel kimondja, hogy a kongruenciarendszernek pontosan egy megoldása van\(\pmod {m}\), és algoritmust is ad ennek kiszámítására. Az egyszerűség kedvéért vezessük be az \(M_i = m / m_i\) jelölést.

Első lépésként keressük meg \(M_i \cdot y_i \equiv 1 \pmod{m_i}\) megoldásait minden \(1 \leq i \leq n\)-re. Ezt átalakíthatjuk a következőképpen: \(y_i = M_i^{-1}\) (\(M_i^{-1}\) alatt \(M_i\) moduláris inverzét értjük az \(m_i\) modulusra).

Az kongruenciarendszer megoldása ekkor \(x\equiv \sum_{i=1}^{n}{y_i \cdot M_i \cdot c_i} \pmod{m}\).

Az algoritmus helyigénye - mindannyiunk meglepetésére - a kongruenciarendszer méretével arányos.

A moduláris inverz kiszámítása\(\pmod {m}\) a szubrutinként felhasznált gyorshatványozás miatt \(\mathcal{O}(\log (m))\) időt vesz igénybe. Ezt összesen \(n\)-szer hajtjuk végre, tehát a teljes algoritmus aszimptotikus futási ideje \(\mathcal{O}(n \cdot \log (m))\) lesz.

Fenwick-fa

A táblázatban atömb, a prefix tömb és a Fenwick-fa különböző műveleteinek komplexitását vetjük össze.
Egyszerű tömb Prefix tömb Fenwick-fa
Inicializálás \(\mathcal{O}(n)\) \(\mathcal{O}(n)\) \(\mathcal{O}(n)\)
Elérés \(\mathcal{O}(1)\) \(\mathcal{O}(1)\) \(\mathcal{O}(\log(n))\)
Prefix lekérés \(\mathcal{O}(n)\) \(\mathcal{O}(1)\) \(\mathcal{O}(\log(n))\)
Módosítás \(\mathcal{O}(1)\) \(\mathcal{O}(n)\) \(\mathcal{O}(\log(n))\)

A Fenwick-fa egy igen hasznos adatszerkezet, amely minden kompetitív programozó alapeszköztárának fontos kelléke. Hogy Sanroy Lozant, a Fenwick-fa tisztelők bizottságának elnökét idézzem:

Programozó, küzdj, és bízva bízzál! Minden megoldásod helyes, ha Fenwick-fát használsz.

Egy olyan statikus adatszerkezetről van szó, ami egyszerre támogat különböző pont-módosításokat és prefix-lekéréseket \(\mathcal{O}(\log(n))\) futási időben. Leggyakoribb alkalmazása a pont-értékmódosítás és az intervallum-összeglekérés.

A magyarázathoz először vezessük be a legjelentéktelenebb bit fogalmát. Egy adott szám legjelentéktelenebb bitje (rövidítve LSB) a szám bináris alakjában a legkisebb helyi értéken álló akív bit.

A binárisan indexelt fa működési elvének alapja, hogy az implementáció során használt tömb egy adott \(i\) indexénél a reprezentált tömb \([i - lsb(i) + 1; i]\) intervallum-összegét tároljuk. Ez teszi lehetővé a módosítás és a lekérés műveletek hatékony végrehajtását.

A pont-módosítás ekkor a következő algoritmus szerint zajlik: 

void update(int index, int value) {
    for (int i = index; i < fenwick_tree.size; i += LSB(i)) {
        fenwick_tree.array[i] += value;
    }
}

A prefix-lekérés pedig: 

int query(int index) {
    int result = 0;
    for (int i = index; i; i -= LSB(i)) {
        result += fenwick_tree.array[i];
    }
    return result;
}

A prefix-lekérésből rögtön adódik az intervallum-lekérés implementációja, felhasználva hogy bármely intervallum-összeg felírható két prefix-összeg különbségeként.

Könnyen látható, hogy a Fenwick-fa kompatibilis minden, az összeadással analóg művelettel, sőt prefix-maximum/minimumlekérésekre is használható. De ne legyünk mohók, feleim; ezekről csak a későbbiekben!
